GEGL -- Gimp 'E' Graphical Library --- is an image processing library based on GtkObjects.  This library will be designed to handle various image processing tasks needed in Gimp initially, but should be useful  to other apps as well.  It will handle color spaces and data types  in a general way, and allow new color models/data types to be added easily. It will include a way of creating chains (or trees) of image operations and evaluating these in a demand-driven way.  Classes for point, area, geometric and statistics operations will be included, as well as color conversion ops for converting images between color models.  Memory management and image classes are planned as well.  It includes a method for writing generic image processing routines (Generic Image Language) and generating specific data type and colormodel code (Codegen) from this.

GEGL is still in a very early phase, and many classes are under construction. There is no official release yet, but you can get it from anonymous cvs.

These documents all describe various parts of gegl. They are included in cvs in the docs directory as well as here.

Getting Gegl

Gegl is available from the gnome cvs site. You can get it through anonymous cvs. There are no official releases yet. To access the tree you need to first set up your CVSROOT enviroment variable so that cvs knows where to go to grab the source from. The proper value for CVSROOT is:

:pserver:anonymous@anoncvs.gnome.org:/cvs/gnome

For example, in csh:

>setenv CVSROOT :pserver:anonymous@anoncvs.gnome.org:/cvs/gnome

In a bourne shell descendant (e.g. bash) the following syntax has to be used :

>export CVSROOT=':pserver:anonymous@anoncvs.gnome.org:/cvs/gnome'

Next, the first time the source tree is checked out, a cvs login is needed.

>cvs login

This will ask you for a password. There is no password so just enter a carriage return. To get the tree and place it in a subdir of your current working directory, issue the command:

>cvs get gegl

You can also browse the cvs source through the web using bonsai.

You will need a recent version of glib and gtk+ and gob. You will also need a lex/yacc (or flex/bison) pair.
